## Project Overview

**abcm2ps** is a C program that converts ABC music notation files to PostScript (PS) or SVG output. It supports multiple voices/staves (the 'm' in the name). Based on abc2ps v1.2.5.

## Build Commands

```sh
# Configure (run once, generates Makefile and config.h)
./configure

# Build
make

# Run tests (converts 10 sample .abc files to .ps and checks output)
make test

# Install
make install

# Clean
make clean
```

### Configuration options
```sh
# Example: set install prefix
./configure --prefix=/usr/local

# Optional: with pango/cairo support for non-Latin text
# (auto-detected via pkg-config)
```

An alternative Ninja build is also available via `build.ninja`.

## Architecture

The pipeline is: **ABC file → parse → layout → render → PS/SVG output**

### Key source files (by role)

| File | Purpose |
|------|---------|
| `abcm2ps.c` | Main entry point, CLI argument parsing, file I/O, memory arena |
| `abcparse.c` | ABC notation parser — converts ABC text to internal symbol representation |
| `front.c` | Front-end: document structure, metadata handling |
| `parse.c` | Format/layout parsing, line breaking, pagination (largest file) |
| `music.c` | Music generation: voices/staves, beams, stems, note positioning |
| `draw.c` | PostScript drawing primitives and output |
| `svg.c` | SVG output generation |
| `deco.c` | Decorations: articulations, ornaments |
| `format.c` | Page layout, margins, font management, `.fmt` file parsing |
| `buffer.c` | PostScript output buffering |
| `subs.c` | Utilities, pango text rendering for international text |
| `syms.c` | PostScript/SVG musical glyph/symbol definitions |
| `glyph.c` | UTF-8 to glyph mapping |

### Headers
- `abcm2ps.h` — main header: all shared data structures, macros, and declarations
- `config.h` — generated by `./configure` (version, platform defines, feature flags)

### Format files (`.fmt`)
Installed to `$(PREFIX)/share/abcm2ps/`. Control rendering style (page layout, fonts, etc.).

### Compile-time features (in `config.h.in`)
- `A4_FORMAT` — European A4 page size default
- `DECO_IS_ROLL` — alternative decoration interpretation
- `HAVE_MMAP` — enable mmap for file reading
- `DEFAULT_FDIR` — default format file search directory

## Testing

Tests convert sample `.abc` files to PostScript and verify output:
```sh
make test
```
Sample files: `sample.abc`, `accordion.abc`, `chinese.abc`, `deco.abc`, `newfeatures.abc`, `voices.abc`, etc.

There is no unit test framework — testing is integration-based via output comparison.

## Notes

- The project is pure C (no C++ or other languages).
- Supports up to 32 simultaneous voices/staves.
- Pango/Cairo integration is optional; detected automatically at configure time.
- `rst2man` is used to generate the man page from `abcm2ps.rst`; if absent, `make install` skips the man page gracefully.
- `config.h` and `Makefile` are generated files — do not edit them directly; edit `config.h.in` and `Makefile.in` instead.
